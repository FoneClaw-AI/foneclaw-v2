# Privacy and Safety

FoneClaw is designed around transparent permissions and user-controlled phone actions.

## Principles

- Permissions should be clear and grouped by purpose.
- Sensitive actions should require user confirmation.
- Users should be able to revoke permissions.
- Product copy should avoid unsupported privacy or security absolutes.
- Phone-control workflows should stay within supported Android actions.

## Sensitive workflows

Messages, calls, purchases, account changes, and system changes should be handled carefully and should not be represented as automatic actions without review.

## Company positioning

FoneClaw is an independent Android product. It is not owned by Xiaomi, Google, Samsung, Apple, or any other phone manufacturer.
